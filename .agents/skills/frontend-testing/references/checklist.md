# 测试生成检查清单

在为 7Flows 前端生成或审查测试时使用此清单。

## 生成前

- [ ] 完整阅读源代码
- [ ] 判断是工具函数、组件、页面还是 hook
- [ ] 确认当前仓库是否已接入前端测试栈
- [ ] 识别要测试的所有文件，而不是只看 `index`
- [ ] 列出核心行为与边界情况

## 测试策略

- [ ] 先测稳定逻辑，再测复杂画布交互
- [ ] 多文件任务一次只处理一个文件
- [ ] 若组件职责过重，先考虑重构
- [ ] 未实现能力要测禁用态或占位态，而不是假设功能已存在

## 必测内容

- [ ] 渲染成功
- [ ] 关键 props 或输入
- [ ] loading / success / error / empty
- [ ] `null` / `undefined` / 空数组 / 空对象
- [ ] 不支持的节点类型或协议类型

## 7Flows 特定检查

- [ ] 节点类型切换是否影响字段显隐
- [ ] 工具/MCP/沙盒/授权开关是否正确显示
- [ ] 调试面板状态是否正确切换
- [ ] 发布协议分支是否正确显示
- [ ] 未落地能力是否处于禁用或实验态

## Mock 检查

- [ ] 只 mock 必要的外部依赖
- [ ] 不伪造当前仓库不存在的全局 mock 体系
- [ ] 每个测试前清理 mocks
- [ ] 共享状态在每个测试前重置

## 生成后

若已接入测试栈：

- [ ] 运行测试
- [ ] 运行 `pnpm lint`
- [ ] 运行 `pnpm exec tsc --noEmit`

若未接入测试栈且本次未要求搭建：

- [ ] 明确说明未运行前端测试
- [ ] 给出后续落测试建议
