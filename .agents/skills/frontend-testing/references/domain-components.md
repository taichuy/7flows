# 领域特定组件测试

本指南覆盖 7Flows 前端中最重要的领域组件测试点。

## 工作流编辑器相关

重点测试：

1. 节点类型切换
2. 节点配置字段显隐
3. 授权来源展示
4. 工具/MCP/沙盒开关
5. 未实现能力的禁用态

### 节点配置面板

重点断言：

- 节点类型变化后 section 是否切换
- 必填字段缺失时是否显示错误
- 非当前节点支持的能力是否隐藏或禁用

## 调试面板相关

重点测试：

1. `pending` / `running` / `succeeded` / `failed`
2. 日志、输入、输出的空态
3. 时间线数据是否正确显示
4. 错误态是否可见

## 发布配置相关

重点测试：

1. `native` / `openai` / `anthropic` 的协议切换
2. 不同协议的 section 显隐
3. 未完成配置时的提示与禁用

## 首页与概览页面

当前仓库还处于骨架期，页面级测试更适合覆盖：

- API 概览成功展示
- 服务状态卡片渲染
- 后端返回异常时的降级展示

## 工具函数与映射函数

最适合优先落测试的领域逻辑包括：

- 状态映射
- 协议映射
- schema 派生
- 节点能力分组
