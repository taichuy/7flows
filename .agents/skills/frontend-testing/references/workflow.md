# 测试工作流指南

本指南定义 7Flows 前端测试的工作流，重点解决“当前仓库测试基础设施还不完整”的现实问题。

## 开始前先判断

先确认本次任务属于哪一种：

1. 仅给测试建议
2. 为现有测试栈补测试
3. 顺手建立第一套前端测试栈并补测试

如果仓库尚未接入测试栈，不要直接写“运行 `pnpm test`”，要先判断是否需要一并搭建。

## 单文件工作流

适用于 `web/lib/*.ts`、简单组件、单个 hook：

1. 完整阅读源文件
2. 列出必须覆盖的行为
3. 若测试栈已存在，编写单测
4. 运行该文件对应测试
5. 修复失败后再继续

## 多文件工作流

当用户要求测试一个目录时：

1. 列出目录下所有待测文件
2. 按复杂度排序：
   - 工具函数
   - hooks
   - 简单组件
   - 容器组件
   - 画布/节点/调试等复杂组件
3. 一次处理一个文件
4. 当前文件通过后再继续下一个

## 何时先重构

若出现以下情况，建议先重构再测：

- 组件职责过多
- 节点卡片里塞了大表单和调试逻辑
- 状态和副作用高度耦合
- 单个文件很难隔离测试

## 完成后的验证

如果仓库已有测试栈：

```bash
pnpm exec vitest run
pnpm lint
pnpm exec tsc --noEmit
```

如果仓库还没有测试栈且本次未要求搭建：

- 只输出建议测试点
- 明确说明尚未执行前端测试
