# 规则目录 — 业务逻辑

## 节点配置必须显式表达授权与运行策略

IsUrgent: True
Category: 业务逻辑

### 描述

7Flows 的节点不是普通表单项。涉及 `llm_agent`、`tool`、`sandbox_code`、`mcp_query`、`router`、`loop`、`output` 等节点时，前端必须显式表达关键业务语义，而不是只保留 prompt 或少数输入框。

至少要关注这些能力是否在 UI 中有可见性：

- 上下文授权来源
- 工具/MCP/沙盒开关
- 输入输出 schema
- 超时、重试或运行策略
- 实验态或未实现状态

### 建议的修复

- 将节点业务配置拆成明确 section，而不是把所有字段摊平成一个大表单。
- 若能力当前尚未落地，使用禁用态或占位说明，而不是假装可用。

## 不要把所有前序节点输出默认暴露给当前节点

IsUrgent: True
Category: 业务逻辑

### 描述

产品文档要求节点间上下文访问是显式授权的。任何 UI 如果默认显示“全部前序节点变量可见”或自动注入所有上游结果，都会误导实现层走向越权共享。

### 建议的修复

- 使用明确的“可读来源”选择或展示。
- 变量选择器只展示已授权范围。
- 对未授权数据给出空态或权限提示，而不是静默兜底为全量可见。

## 未实现能力必须诚实表达

IsUrgent: False
Category: 业务逻辑

### 描述

当前 7Flows 仍处于早期阶段，完整 loop 执行、插件代理、MCP 查询、发布协议映射等能力并未全部落地。前端如果直接展示为“已配置完成”，会造成错误预期。

### 建议的修复

- 使用 `coming soon`、`experimental`、禁用态或只读态。
- 不要提供会让用户以为后端已支持的交互入口。
