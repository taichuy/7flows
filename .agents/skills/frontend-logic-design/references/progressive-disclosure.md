# 渐进式披露（Progressive Disclosure）

## 定义

渐进式披露是一种交互设计模式：**不一次展示所有信息，而是分层展示，用户通过主动行为触发下一层**。

来源：Nielsen Norman Group
核心数据：减少界面上 50% 的选项，任务完成率提升 20-30%。

## 核心原则

### 1. 分层展示

将信息分为 3-4 个深度层：

| 层级 | 可见性 | 内容 | 目标用户 |
|------|--------|------|---------|
| Essential（核心） | 始终可见 | 80% 用户 80% 时间需要的内容 | 所有人 |
| Standard（标准） | 一次交互可达 | 常用但非核心的功能 | 多数用户 |
| Advanced（高级） | 两次以上交互可达 | 专家功能、高级配置 | 少数用户 |

### 2. 主动触发

层级之间的切换必须由用户主动触发，不能自动跳转：
- 点击展开
- 点击导航
- 点击"查看更多"
- 悬停显示（仅限桌面端）

### 3. 入口暗示

**每个被隐藏的功能必须有可见的入口暗示其存在**：
- Chevron（>）暗示可展开
- "查看全部 →" 暗示有更多内容
- Badge 数字暗示有待处理的项目
- "高级设置" 文字暗示有更多选项

## 常见 UI 模式

| 模式 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **手风琴（Accordion）** | 同一页面内展开详情 | 保持上下文，干扰小 | 内容位移可能迷失 |
| **Tab 标签页** | 平行分类的同级内容 | 分类清晰，可扫描 | 隐藏内容可发现性低 |
| **侧栏抽屉（Drawer）** | 主内容旁的补充详情 | 保持上下文，空间充足 | 移动端空间受限 |
| **弹窗（Modal）** | 需要聚焦注意力的确认 | 聚焦明确，不可忽略 | 打断流程，阻塞背景 |
| **下钻导航（Drill-down）** | 概览到详情的递进 | 清晰的层级路径 | 需要面包屑防止迷失 |
| **"更多"菜单** | 低频操作 | 保持界面整洁 | 可发现性极低 |

## 反模式

### 1. 过度隐藏（Over-hiding）

关键功能或主要操作被埋得太深，用户无法发现。

```
❌ 把"创建新项目"放在三级菜单里
✅ "创建新项目"在项目列表页面直接可见
```

### 2. 层级过深

超过 3 层的渐进式披露会让用户迷失方向。

```
❌ 看板 → 卡片 → 侧栏 → 弹窗 → 又一个弹窗
✅ 看板 → 侧栏（或）看板 → 独立页面
```

### 3. 不一致的披露模式

同一页面内，相似的元素使用不同的披露方式。

```
❌ 卡片A点击弹出侧栏，卡片B点击弹出弹窗，卡片C点击跳转页面
✅ 所有列表型卡片点击条目统一弹出侧栏
```

### 4. 没有回退路径

用户深入到下一层后，没有明确的方式回到上一层。

```
❌ 点开详情后没有关闭按钮，也没有面包屑
✅ 侧栏有明确的关闭按钮，页面有面包屑导航
```

## 与 Shneiderman's Mantra 的关系

Ben Shneiderman 的信息可视化法则：

> "Overview first, zoom and filter, then details on demand."

这本质上就是渐进式披露在数据可视化领域的应用：

| Shneiderman | 渐进式披露 | 实际应用 |
|-------------|----------|---------|
| Overview | Essential layer | 看板卡片：Top N 摘要 |
| Zoom & Filter | Standard layer | 独立页面：全量数据 + 搜索筛选 |
| Details on Demand | Advanced layer | 侧栏/详情页：单条记录完整信息 |

## 在看板/仪表盘中的特殊考虑

### 信息密度平衡

看板天然是"概览层"，信息密度要高但不能过载。经验法则：
- 每张卡片 3-7 条数据项
- 每条数据项不超过一行
- AI 建议/摘要最多一句话

### 刷新 vs 重新生成

对于 AI 生成的内容，"刷新"按钮的语义要清晰：
- **刷新**（普通加载）= 从缓存取最新数据
- **重新生成**（force）= 跳过缓存，AI 重新计算

用户需要知道哪种刷新正在发生。建议：普通加载用加载骨架屏，强制重新生成用明确的按钮 + loading 状态文案。
