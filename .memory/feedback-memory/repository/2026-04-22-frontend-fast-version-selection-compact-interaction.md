# 快速选择版本这类交互应优先采用紧凑直达设计

## 时间

`2026-04-22 08`

## 规则

- 当页面目标是“快速选择版本”这类窄而高频的交互时，优先采用紧凑直达设计，不要再绕到额外“管理”按钮和独立管理弹窗。
- 如果当前值已经通过控件本身清楚显示，例如下拉框当前选中的版本号，就不要再叠加“当前”标签、禁用态按钮或重复说明。
- 在同一行里，优先使用“当前值选择器 + 辅助动作按钮”的组合，例如“版本选择 + 更新”。
- 按钮文案在组件上下文已经足够明确时应保持简洁，优先使用“添加”“删除”“配置”这类短词，不要无必要写成长文案。

## 原因

- 当用户只是想快速切换版本时，如果还要绕到额外“版本管理”按钮和弹窗，会打断操作路径，也会把本来很窄的交互做重。
- 当前值已经显示时，再叠加“当前”标签或禁用态，会造成语义重复，视觉上也更臃肿、更难看。
- 在按钮已经位于明确组件上下文里时，过长文案只会增加噪音；简洁词更容易扫读，也更符合后台操作面板的节奏。

## 适用场景

- 设置页、控制台页、管理台页中的快速切换版本或快速选择当前值的交互设计。
- 已有明确组件上下文，用户不需要依赖长文案才能理解按钮含义。
- 需要在紧凑表格或列表里同时承载切换、升级、配置等动作。
