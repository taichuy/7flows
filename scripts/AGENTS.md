# scripts 协作说明

先读根目录 `AGENTS.md` 与 `docs/dev/team-conventions.md`，再处理 `scripts/`。

## 目录定位

- `scripts/` 只放开发、验证、治理与维护辅助脚本，不承载产品运行时主链。
- 脚本默认服务本地开发、CI 或仓库治理，不要把远程运维、人工通知或外部平台耦合成共享默认入口。

## 默认约束

- 保持 local-first、loopback-first；拒绝 `curl | bash`、隐藏下载、静默上传、外部 webhook 或凭证外传。
- 脚本的副作用要显式：写哪些文件、会启动哪些进程、是否可能联网，都要让调用者看得见。
- 新增启动脚本优先复用仓库现有目录约定，例如 `docker/`、`api/`、`web/`、`tmp/logs/`，不要偷偷写到用户家目录。
- 默认提供清晰帮助信息；如支持后台进程，优先给出 `status` / `stop` 一类可恢复入口。
- 修改脚本后至少做与类型匹配的验证；Shell 脚本至少通过 `bash -n` 与最小无副作用参数验证。

## 文档同步

- 新增、删除或显著改变共享脚本时，同步更新 `README.md` 中的使用入口。
- 如果脚本引入新的执行边界、依赖前提或安全假设，也要在本目录 `AGENTS.md` 或相关 ADR 里补说明。
