# Docker 本地运行态目录要进 gitignore

## 时间

`2026-04-12 16`

## 规则

- 与 Docker 本地运行态相关的目录和本地环境文件需要补进仓库根 `.gitignore`。

## 原因

- 这些目录和文件属于本机运行产物，容易因为权限、容器写入和环境差异产生噪音，不适合进入版本控制。

## 适用场景

- 新增 Docker 中间件目录。
- Docker 运行过程中生成本地 volumes、环境文件或其他宿主机状态目录时。

## 分类

- 团队协作偏好

## 备注

- 本次已将 `docker/volumes/` 和常见 Docker 本地环境文件入口补进根 `.gitignore`。
