# 受 VoltAgent 启发的设计系统

## 1. 视觉主题与氛围

VoltAgent 的界面是一个面向 AI 时代的深空命令终端——一个面向开发者的暗色界面，构建在近乎纯黑的表面（`#050507`）之上，唯一的视觉打断是翠绿色能量那如同电脉冲般的闪耀。整个体验让人感觉像是在凌晨两点凝视一个高功率的 IDE：黑暗、专注、充满使命感。这不是一个友好的 SaaS 落地页——它是一个通过代码片段、架构图和技术自信来彰显自身的工程平台。

绿色强调色（`#00d992`）被精准地使用——它在标题、边框和交互元素上发光，如同承载信号的电路板。在碳黑色的画布上，这种绿色读作“已通电”——这是针对 AI Agent 工程平台的有意视觉隐喻。辅助调色板完全由暖中性灰（`#3d3a39`、`#8b949e`、`#b8b3b0`）构成，这些颜色柔化了黑暗感而不引入色彩噪音，创造出一种纯蓝灰色所缺乏的驾驶舱般的温暖感。

排版上，标题采用系统字体栈——以实现最快的渲染速度和原生般的权威感——而 Inter 字体则以几何精度承载正文和 UI 文本。代码块使用 SFMono-Regular，这是开发者在终端中看到的相同字体，在每一次滚动中强化了工具的可信度。

**主要特征：**
- 碳黑画布（`#050507`）搭配暖灰边框（`#3d3a39`）——不冰冷，不苍白
- 单一强调色：翡翠信号绿（`#00d992`）作为唯一的色彩能量来源
- 双字体系统：system-ui 用于权威标题，Inter 用于精确 UI/正文，SFMono 用于代码可信度
- 超紧致标题行高（1.0–1.11）创造出密集、压缩的力量块
- 暖中性调色板（`#3d3a39`、`#8b949e`、`#b8b3b0`）防止深色主题显得过于冷淡
- 开发者终端美学，其中代码片段本身就是英雄内容
- 绿色发光效果（`drop-shadow`、边框强调）使 UI 元素感觉像通电一般

## 2. 调色板与角色

### 主色
- **翡翠信号绿**（`#00d992`）：核心品牌能量——用于强调边框、发光效果和最高信号的交互时刻。这是整个界面的“通电”指示器。
- **VoltAgent 薄荷绿**（`#2fd6a1`）：品牌绿色的按钮文本变体——比纯信号绿略暖且更具可读性，专门用于深色表面上的 CTA 文本。
- **Tailwind 祖母绿**（`#10b981`）：生态系统标准的绿色，以低透明度（30%）用于微妙的背景色调和链接默认状态。在 VoltAgent 的自定义调色板与 Tailwind 工具类之间架起桥梁。

### 次要色与强调色
- **柔紫**（`#818cf8`）：一种冷静的靛蓝紫，少量用于次要分类、代码语法高亮和视觉多样性，而不与绿色竞争。
- **钴蓝主色**（`#306cce`）：Docusaurus 主深色——在文档上下文中用于链接和交互焦点状态。
- **深钴蓝**（`#2554a0`）：最深的原色，保留用于文档 UI 中的按下/活动状态。
- **环蓝**（`#3b82f6`）：Tailwind 的环颜色，透明度 50%——仅在键盘焦点期间可见，以确保无障碍合规。

### 表面与背景
- **深渊黑**（`#050507`）：落地页画布——一种近乎纯黑、带有最微弱暖色调的颜色，比大多数“深色主题”更深，以最大化与绿色强调色的对比。
- **碳表面**（`#101010`）：主要卡片和按钮背景——比深渊黑浅一个色阶，创造出几乎不可察觉的抬高层。用于所有包含的表面。
- **暖炭灰边框**（`#3d3a39`）：标志性的容器颜色——不是冷灰色，而是一种温暖、近乎棕色的暗色调，防止边框在黑色画布上显得刺眼。

### 中性色与文本
- **雪白**（`#f2f2f2`）：深色表面上的主要文本颜色——不是纯白色（`#ffffff`），而是一种柔和、护眼的米白色。网站上使用最多的颜色（1008 次）。
- **纯白**（`#ffffff`）：保留给最高强调的时刻——幽灵按钮文本和最高对比度的标题。以低透明度（5%）用于微妙的叠加效果。
- **暖羊皮纸**（`#b8b3b0`）：次要正文文本——一种带有轻微粉调的暖浅灰色，在深色画布上读起来像“纸张”。
- **钢石板灰**（`#8b949e`）：三级文本、元数据、时间戳和弱化内容。一种冷蓝灰色，在暖羊皮纸之下提供清晰的层级。
- **雾灰**（`#bdbdbd`）：页脚链接和辅助导航文本——悬停时变亮至纯白。
- **薄雾灰**（`#dcdcdc`）：比雾灰稍亮，用于次要链接文本，悬停时过渡到亮绿色。
- **近白**（`#eeeeee`）：最高对比度的次要文本，仅次于雪白。

### 语义与强调色
- **成功祖母绿**（`#008b00`）：深绿色，用于文档上下文中的成功状态和正向确认。
- **成功浅色**（`#80d280`）：柔和的淡绿色，用于成功背景和微妙的正面指示。
- **警告琥珀**（`#ffba00`）：亮琥珀色，用于警告提醒和谨慎状态。
- **警告淡色**（`#ffdd80`）：柔化的琥珀色，用于警告背景填充。
- **危险珊瑚**（`#fb565b`）：鲜艳的红色，用于错误状态和破坏性操作警告。
- **危险玫瑰**（`#fd9c9f`）：柔化的珊瑚粉，用于错误背景。
- **信息蓝绿**（`#4cb3d4`）：冷蓝绿色，用于信息提示和技巧提示。
- **虚线边框石板灰**（`#4f5d75` 透明度 40%）：一种柔和的蓝灰色，专门用于工作流图中的装饰性虚线边框。

### 渐变系统
- **绿色信号发光**：`drop-shadow(0 0 2px #00d992)` 动画过渡到 `drop-shadow(0 0 8px #00d992)`——在 VoltAgent 闪电标志和交互元素上产生脉动的“电荷”效果。发光像心跳一样膨胀和收缩。
- **温暖环境雾**：`rgba(92, 88, 85, 0.2) 0px 0px 15px`——一种暖色调的漫射阴影，在高架卡片周围产生柔和的大气辉光，边缘可见而无锐利边界。
- **深度戏剧性抬升**：`rgba(0, 0, 0, 0.7) 0px 20px 60px` 配合 `rgba(148, 163, 184, 0.1) 0px 0px 0px 1px inset`——一种沉重、戏剧性的向下阴影，搭配微弱的嵌入式石板环，用于最突出的浮动元素。

## 3. 字体排版规则

### 字体系列
- **主字体（标题）**：`system-ui`，后备字体：`-apple-system, Segoe UI, Roboto, Ubuntu, Cantarell, Noto Sans, Helvetica, Arial, Apple Color Emoji, Segoe UI Emoji, Segoe UI Symbol`
- **次要字体（正文/UI）**：`Inter`，后备字体继承自 system-ui 堆栈。OpenType 特性：`"calt", "rlig"`（上下文替代和必需连字）
- **等宽字体（代码）**：`SFMono-Regular`，后备字体：`Menlo, Monaco, Consolas, Liberation Mono, Courier New, monospace`

### 原则
- **系统原生权威**：展示标题使用 system-ui 而非自定义网页字体——这意味着最大的文本会立即渲染（无 FOIT/FOUT）并继承操作系统的原生特性。在 macOS 上是 SF Pro，在 Windows 上是 Segoe UI。设计将这种可变性视为特性，而非缺陷。
- **紧致压缩创造密度**：英雄行高极度压缩（1.0）并配合负字间距（-0.65px 到 -0.9px），创造出感觉像密集技术规范而非松散营销文案的文本块。
- **权重梯度，而非权重对比**：系统使用温和的 300→400→500→600→700 权重递进。粗体（700）保留给副标题和代码按钮强调。大多数正文文本位于 400–500，创造微妙而非戏剧性的层级。
- **大写是挣来的且宽松的**：当使用大写时，总是配合慷慨的字间距（0.45px–2.52px），将密集的单词转换为分散的上划线标签。这种处理绝不应用于标题。
- **默认启用 OpenType**：system-ui 和 Inter 都启用了 `"calt"` 和 `"rlig"` 特性，确保整个过程中上下文字符调整和连字渲染。

## 4. 组件样式

### 按钮

**幽灵/线框（标准）**
- 背景：透明
- 文本：纯白（`#ffffff`）
- 内边距：舒适（12px 16px）
- 边框：细实线暖炭灰（`1px solid #3d3a39`）
- 圆角：舒适圆角（6px）
- 悬停：背景变暗至 `rgba(0, 0, 0, 0.2)`，透明度降至 0.4
- 轮廓：微妙的绿色调（`rgba(33, 196, 93, 0.5)`）
- 默认交互元素——低调但明显可点击

**主要绿色 CTA**
- 背景：碳表面（`#101010`）
- 文本：VoltAgent 薄荷绿（`#2fd6a1`）
- 内边距：舒适（12px 16px）
- 边框：无可见边框（基于轮廓的焦点指示器）
- 轮廓：VoltAgent 薄荷绿（`rgb(47, 214, 161)`）
- 悬停：与幽灵按钮相同的变暗行为
- “已通电”按钮——深色表面上的绿色文本读起来像活动终端命令

**第三级/强调容器按钮**
- 背景：碳表面（`#101010`）
- 文本：雪白（`#f2f2f2`）
- 内边距：慷慨（四周 20px）
- 边框：粗实线暖炭灰（`3px solid #3d3a39`）
- 圆角：舒适圆角（8px）
- 一种类似卡片的按钮处理方式，用于较大的交互表面（代码复制块、功能 CTA）

### 卡片与容器
- 背景：碳表面（`#101010`）——比页面画布浅一个色阶
- 边框：标准容器使用 `1px solid #3d3a39`（暖炭灰）；高亮/活动卡片使用 `2px solid #00d992`
- 圆角：内容卡片舒适圆角（8px）；较小的内联容器微圆角（4–6px）
- 阴影层级 1：温暖环境雾（`rgba(92, 88, 85, 0.2) 0px 0px 15px`）用于标准抬升
- 阴影层级 2：深度戏剧性（`rgba(0, 0, 0, 0.7) 0px 20px 60px` + `rgba(148, 163, 184, 0.1) 0px 0px 0px 1px inset`）用于英雄/特色展示卡片
- 悬停行为：边框颜色可能向绿色强调色转变，或透明度轻微增加
- 虚线变体：`1px dashed rgba(79, 93, 117, 0.4)` 用于工作流/图表容器——视觉上区别于实线边框的内容卡片

### 输入框与表单
- 未提取到明确的输入框令牌数据——该网站以落地页为主，表单 UI 极少
- npm 安装命令（`npm create voltagent-app@latest`）以代码块而非输入框形式呈现
- 推断样式：碳表面背景，暖炭灰边框，VoltAgent 薄荷绿焦点环，雪白文本

### 导航
- 深渊黑画布上的粘性顶部导航栏
- Logo：VoltAgent 闪电图标，带有动画绿色发光（`drop-shadow` 在 2px–8px 之间循环）
- 导航结构：Logo → 产品下拉 → 用例下拉 → 资源下拉 → GitHub stars 徽章 → Docs CTA
- 链接文本：雪白（`#f2f2f2`），Inter 14–16px，字重 500
- 悬停：链接过渡到绿色变体（`#00c182` 或 `#00ffaa`）
- GitHub 徽章：社交证明元素直接集成到导航中
- 移动端：折叠为汉堡菜单，单列垂直布局

### 图像处理
- 深色主题的产品截图和架构图占主导地位
- 代码块被视为主要视觉内容——使用 SFMono-Regular 进行语法高亮
- Agent 工作流可视化呈现为带有绿色连接线的交互式节点图
- 装饰性点状背景出现在英雄区域后面
- 在卡片容器内全出血，保持 8px 圆角

### 特色组件

**npm 安装命令块**
- 突出的代码片段（`npm create voltagent-app@latest`），样式为可复制命令
- 碳表面上的 SFMono-Regular，带复制到剪贴板按钮
- 作为主要 CTA——“先安装，后阅读”的开发者心理

**公司 Logo 滚动条**
- 水平滚动的开发者/公司 Logo 条
- 无限动画（`scrollLeft`/`scrollRight`，持续时间 25–80 秒）
- 悬停时暂停，并为偏好减少动效的用户暂停
- 展示生态系统采用情况，而不使布局杂乱

**功能区域卡片**
- 结合代码示例和描述性文本的大卡片
- 左侧：带语法高亮的代码片段；右侧：功能描述
- 高亮/活动功能上使用绿色强调边框（`2px solid #00d992`）
- 内部内边距：慷慨（估计 24–32px）

**Agent 流程图**
- 显示 Agent 协作的交互式节点图可视化
- 连接线使用 VoltAgent 绿色变体
- 节点样式为暖炭灰边框系统内的小型卡片

**社区 / GitHub 区域**
- 大型 GitHub 图标作为视觉锚点
- 突出显示星标数量和贡献者指标
- 温暖的社交证明：页脚中的 Discord、X、Reddit、LinkedIn、YouTube 链接

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 标尺：2px、4px、5px、6px、6.4px、8px、12px、16px、20px、24px、28px、32px、40px、48px、64px
- 按钮内边距：12px 16px（标准），20px（容器按钮）
- 卡片内部内边距：大约 24–32px
- 区域垂直间距：慷慨（估计主要区域之间 64–96px）
- 组件间隙：兄弟卡片/元素之间 16–24px

### 网格与容器
- 最大容器宽度：大约 1280–1440px，居中
- 英雄区域：居中单列，最大呼吸空间
- 功能区域：交替的非对称布局（左代码右文本，然后反转）
- Logo 滚动条：全宽水平滚动，打破容器约束
- 卡片网格：功能展示使用 2–3 列
- 集成网格：用于合作伙伴/集成图标的响应式多列

### 留白哲学
- **区域之间的电影级呼吸空间**：巨大的垂直间隙创造了“滚动浏览章节”的体验——每个区域都像一个新的场景。
- **组件内部紧密**：卡片和代码块内部紧凑，行高紧凑，内边距可控。信息是集中的，而不是分散的。
- **边框定义的分离**：VoltAgent 不仅仅依赖留白，还使用暖炭灰边框系统（`#3d3a39`）来划定内容区域。边框本身就是留白信号。
- **英雄优先的层级**：页面顶部占据最多的空间——“AI Agent 工程平台”标题和 npm 命令在第一个内容区域出现之前获得最大的垂直空间。

### 圆角标尺
- 近乎方形（4px）：小型内联元素、SVG 容器、代码跨度——最锐利的处理，传达技术精度
- 微圆角（6px）：按钮、链接、剪贴板操作——交互元素的主力圆角
- 代码专用（6.4px）：代码块、`pre` 元素、剪贴板复制目标——与标准 6px 有意的微小区别
- 舒适圆角（8px）：内容卡片、功能容器、强调按钮——标准容器圆角
- 药丸形（9999px）：标签、徽章、状态指示器、药丸形导航元素——用于小型分类标签的最圆润处理

| 强调色（层级 3） | `2px solid #00d992`，无阴影 | 活动/高亮功能卡片，选中状态 |
| 环境辉光（层级 4） | `rgba(92, 88, 85, 0.2) 0px 0px 15px` | 高架卡片，悬停状态，柔和的大气抬升 |
| 戏剧性浮动（层级 5） | `rgba(0, 0, 0, 0.7) 0px 20px 60px` + `rgba(148, 163, 184, 0.1) 1px inset` | 英雄功能展示，模态框，最大高程内容 |

**阴影哲学**：VoltAgent 主要通过**边框粗细和颜色**来传达深度，而不是阴影。标准的 `1px solid #3d3a39` 边框就是高程——增加 `3px` 边框粗细或切换到绿色（`#00d992`）比添加阴影更能传达重要性。当使用阴影时，它们要么是温暖而漫射的（层级 4），要么是电影级而戏剧性的（层级 5）——从不使用中等或通用阴影。

### 装饰性深度
- **绿色信号发光**：VoltAgent 闪电标志以翡翠信号绿在 2px 和 8px 模糊半径之间循环的 `drop-shadow` 动画脉动。这是最独特的装饰元素——它使标志感觉“已通电”。
- **暖炭灰容器线**：`#3d3a39` 边框的暖色调在冷黑色背景上创造出微妙的视觉温暖，仿佛卡片从内部微微发热。
- **虚线工作流线**：`1px dashed rgba(79, 93, 117, 0.4)` 为架构图创造出蓝图般的美感，视觉上区别于实线内容边框。

## 7. 应做与不应做

### 应做
- 使用深渊黑（`#050507`）作为落地页背景，对所有包含的元素使用碳表面（`#101010`）——双色深色系统至关重要
- 翡翠信号绿（`#00d992`）仅用于高信号时刻：活动边框、发光效果和最重要的交互强调
- 深色表面上的按钮文本使用 VoltAgent 薄荷绿（`#2fd6a1`）——比纯信号绿更具可读性
- 保持标题行高压缩（1.0–1.11）并配合负字间距，以形成密集、权威的文本块
- 使用暖灰色调色板（`#3d3a39`、`#8b949e`、`#b8b3b0`）用于边框和次要文本——温暖感防止深色主题显得过于冷淡
- 将代码片段作为主要内容呈现——它们是英雄元素，而非辅助插图
- 使用边框粗细（1px → 2px → 3px）和颜色变化（`#3d3a39` → `#00d992`）来传达深度和重要性，而非依赖阴影
- 标题使用 system-ui，正文使用 Inter——原生字体的速度/权威性与几何无衬线的精度相结合
- 所有代码内容使用 SFMono-Regular——这是开发者可信度的信号
- 在所有文本上应用 `"calt"` 和 `"rlig"` OpenType 特性

### 不应做
- 不要使用明亮或浅色背景作为主要表面——整个身份建立在近黑色之上
- 不要引入暖色（橙、红、黄）作为装饰性强调色——调色板严格限定为绿色 + 黑色上的暖中性色。暖色仅保留给语义状态（警告、错误）
- 不要在大面积表面或背景填充上使用翡翠信号绿（`#00d992`）——它是强调色，绝不是表面色
- 不要将标题行高增加到 1.33 以上——压缩密度是工程平台身份的核心
- 不要大量使用重阴影——深度来自边框处理，而非 box-shadow。阴影仅保留给层级 4–5 的抬升
- 不要使用纯白（`#ffffff`）作为默认正文文本——雪白（`#f2f2f2`）是标准。纯白保留给最高强调的标题和按钮文本
- 不要混入衬线或装饰性字体——整个系统是几何无衬线 + 等宽字体
- 不要在内容卡片上使用大于 8px 的圆角——9999px（药丸形）仅用于小型标签和徽章
- 不要跳过暖灰边框系统——没有 `#3d3a39` 边框的卡片会失去容器感，在深色画布上漂浮不定
- 不要过度动画——动画应缓慢而微妙（滚动条持续时间 25–100 秒，温和的发光脉冲）。快速动画与“工程精度”的氛围相矛盾

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 主要变化 |
|------|------|----------|
| 小移动端 | <420px | 最小布局，全部堆叠，英雄文本缩小至约 24px |
| 移动端 | 420–767px | 单列，汉堡菜单，全宽卡片，英雄文本约 36px |
| 平板端 | 768–1024px | 开始 2 列网格，紧凑导航，中等英雄文本 |
| 桌面端 | 1025–1440px | 完整多列布局，带下拉菜单的扩展导航，大英雄文本（60px） |
| 大桌面端 | >1440px | 最大宽度容器居中（估计 1280–1440px），慷慨的水平边距 |

*共检测到 23 个断点，范围从 360px 到 1992px——表明这是一个流畅、高度响应式的网格系统，而非固定的断点吸附。*

### 触摸目标
- 按钮使用舒适内边距（最小 12px 16px）确保足够的触摸区域
- 导航链接间距足够，便于拇指导航
- 交互式卡片表面足够大，可作为完整的触摸目标
- 建议最小触摸目标：44x44px

### 折叠策略
- **导航**：带下拉菜单的全水平导航在移动端折叠为汉堡菜单
- **功能网格**：3 列 → 2 列 → 单列垂直堆叠
- **英雄文本**：60px → 36px → 24px 渐进缩放，保持压缩比例
- **Logo 滚动条**：调整滚动速度和项目大小；保持无限循环
- **代码块**：在较小视口上水平滚动而非换行——保持代码可读性
- **区域内边距**：按比例减少，但在章节之间保持慷慨的垂直节奏
- **卡片**：移动端垂直堆叠，全宽处理，保持内部内边距

### 图像行为
- 深色主题的截图和图表在容器内按比例缩放
- Agent 流程图在窄视口上简化或水平滚动
- 点状装饰性背景随视口缩放
- 断点之间无明显的艺术指导变化——相同的裁剪，按比例缩放
- 折叠下方图像使用懒加载（Docusaurus 默认行为）
