# 背景
userDocs开发者私有目录，主要是记录AI与开发者之间约定和协作边界，所以在于用户交互时候请先进行阅读对话。
如果还是空请先与用户沟通进行沟通初始化
## 存什么？
记忆目的不是为了给用户画像，而是更好帮助这个人，并且所有日期应该转为绝对日期格式：yyyy-mm-dd hh
### 用户记忆
用户是谁？角色？技术背景？工作习惯？知识水平？
在与用户交互过程通过上述内容就可以，调整与用户沟通方式
### 反馈记忆
当用户给出指正和最终同意时候应该将对应反馈整理为反馈记忆
反馈记忆应该遵守下列规则：
1.每一条都应该包含三部分：规则原因，适用场景
2.记忆纠正，也记忆肯定，防止agent越来越保守，也能保证agent在新场景下直接复用
3.区分偏好，个人和项目团队开发要进行区分
### 项目记忆
1.谁在做什么？
2.为什么这样做？
3.为什么要做？
4.截止日期
5.决策背后动机
### 引用记忆
这里主要是用户给文件资源目录参考文档
## 不存放什么
1.代码模式/架构/路径，可以直接在项目实际代码中获取不需要记录
2.版本管理历史，每次工作结束都会提交代码，什么直接看git就可以了。
3.调试方案/修复方法
4.配置文件已有的
6.临时性任务细节

## 目录索引
- `docs/userDocs/user-memory.md`
  - 用户记忆主文件，包含角色、背景、习惯、知识水平与用户偏好。
- `docs/userDocs/feedback-memory/`
  - 反馈记忆目录，存放可复用的反馈规则；当前保留模板文件。
- `docs/userDocs/project-memory/`
  - 项目记忆目录，存放当前项目共识、阶段目标、责任与动机；当前保留模板文件。
- `docs/userDocs/reference-memory/`
  - 引用记忆目录，存放源码参考、API 查看、脚本相关等引用分类说明与索引。
- `docs/userDocs/todolist/`
  - ai自主开发模式下，一定要用户离线情况，待办确认目录，处理之后直接删除不做沉淀
