# 背景

`docs/userDocs` 是开发者私有目录，主要记录 AI 与开发者之间的约定、协作边界与记忆沉淀。

与用户交互时，应先阅读本目录相关文档，再进行对话或执行后续工作。

## 存什么？

记忆目的不是为了给用户画像，而是为了更好地帮助这个人协作开发。

所有日期应统一转为绝对日期格式：`yyyy-mm-dd hh`。

### 用户记忆

用户记忆用于帮助 AI 调整沟通与协作方式，重点记录：

1. 用户是谁、承担什么角色。
2. 用户的技术背景。
3. 用户的工作习惯。
4. 用户的知识水平。
5. 用户偏好。

### 反馈记忆

反馈记忆应遵守下列规则：

1. 每一条都应包含三部分：规则、原因、适用场景。
2. 既记忆纠正，也记忆肯定，防止 agent 越来越保守，也便于在新场景下直接复用。
3. 区分个人偏好与项目/团队协作偏好。

### 项目记忆

项目记忆重点记录：

1. 谁在做什么。
2. 为什么这样做。
3. 为什么要做。
4. 截止日期。
5. 决策背后的动机。

### 引用记忆

引用记忆主要存放用户提供的参考资料、目录、源码、接口查看入口与脚本说明等引用信息。

## 不存放什么

1. 代码模式、架构、路径，这些内容可以直接从项目实际代码中获取，不需要重复记录。
2. 版本管理历史，每次工作结束都会提交代码，直接看 Git 即可。
3. 调试方案、修复方法。
4. 配置文件中已经明确存在的信息。
5. 临时性任务细节。

## 目录索引

- `docs/userDocs/user-memory.md`
  - 用户记忆主文件，包含角色、背景、习惯、知识水平与用户偏好。
- `docs/userDocs/feedback-memory/`
  - 反馈记忆目录，存放可复用的反馈规则；当前保留模板文件。
- `docs/userDocs/project-memory/`
  - 项目记忆目录，存放当前项目共识、阶段目标、责任与动机；当前保留模板文件。
- `docs/userDocs/reference-memory/`
  - 引用记忆目录，存放源码参考、API 查看、脚本相关等引用分类说明与索引。
