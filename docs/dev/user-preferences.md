# 用户偏好与自治开发偏好

本文记录当前仍稳定成立、并且值得让后续 AI 会话持续复用的用户偏好与自治开发偏好。

## 共享文档偏好

- 当前共享仓库中的重点文档、skills、ADR 与新增治理条目默认使用中文。
- 如果关键入口文档里出现英文且已经影响搜索、检索或后续 AI 理解，应优先翻回中文，而不是继续叠加中英混写。
- 公共文档优先保留稳定规则、当前事实和可持续复用的判断，不要把按日期的个人开发过程继续堆成共享知识库。
- 当前开发者自己的按日期留痕、临时推导和本地记忆放在 `docs/.private/history/` 或 `docs/.private/` 其他子目录，不进入共享仓库。

## 自治开发偏好

- AI 应作为 7Flows 的持续迭代开发者，目标是推进整体完整度、主链路闭环和长期可演进性，而不是做局部美化。
- 每轮开始前默认先读取：`AGENTS.md`、本文、`docs/product-design.md`、`docs/technical-design-supplement.md`、`docs/dev/runtime-foundation.md`、`docs/open-source-commercial-strategy.md`，再检查 `git status`。
- 如果任务命中仓库 skill，优先复用 `.agents/skills/*/SKILL.md`；如果任务涉及协作守则、审查或合并，再补读 `docs/dev/team-conventions.md`。
- 如果当前本地开发者需要衔接最近几轮的个人过程记忆，可再查看 `docs/.private/history/`，但不要把它当成共享事实来源。

## 好项目 / 好代码 / 好架构

- 好项目：主链路清晰且逐步闭环；用户层、AI 与人协作层、AI 治理层持续向产品目标收敛；架构、文档、运行基础和代码结构都能支撑后续持续迭代；对人和对 AI 都具备可理解、可维护、可检索、可追踪性。
- 好代码：首先正确，其次易懂、易改、边界清晰、可测试；单一职责按变化原因判断；解耦目标是切断变化传播路径；业务规则不要直接耦合数据库字段、SDK 细节、第三方返回格式和展示文案；对外契约稳定，内部实现允许演化。
- 好架构：稳定核心承载业务规则，变化实现收敛在边界和适配层；业务层决定“该不该做”，适配层处理“怎么做到”；外部变化不应直接污染核心规则；关键模型、关键状态和关键业务动作应有清晰唯一入口。

## 坏信号

- 长时间停在局部修补、样式整理、代码洁癖式重构，而主链闭环没有推进。
- 架构复杂度上升，但没有增强扩展性、兼容性、可靠性、稳定性或安全性。
- 文件职责混杂、边界泄漏、内部实现直接暴露为外部契约。
- 一个改动需要同步修改过多模块，说明变更传播过大。
- 过度拆分导致碎片化，或过度聚合导致理解和维护困难。
- 只优化技术局部，而没有服务于产品目标和场景完整性。

## 单轮工作偏好

- 默认优先参考 `docs/dev/runtime-foundation.md` 的当前优先级，选择本轮最高优先级任务。
- 单轮只推进一个主主题，最多附带一个为主主题服务的结构调整。
- 优先解决阻碍功能推进、阻碍扩展、阻碍可靠运行的问题；只有当架构问题已经成为阻塞时，才优先做架构增强。
- 如果业务关键场景仍未闭环，功能补全优先于非阻塞型重构。
- 文件解耦不要按长度机械拆分；重点看变化原因是否混杂、边界是否泄漏、改动传播是否过大、是否影响维护/扩展/检索/追踪，以及是否同时对人和 AI 不友好。
- 文件长度只作为预警：后端超过 1500 行重点关注，前端超过 2500 行重点关注。
- 如果判断项目已经进入需要人工逐项进行界面设计的阶段，再运行：

```text
node "e:\\code\\taichuCode\\ai-presson-wen\\aionui\\notice\\kill-port-8050.js"
```

## 每轮汇报要求

每轮结束后，默认汇报以下内容：

1. 当前项目的主要阶段问题是什么。
2. 为什么选择本轮任务，而没有优先做其他事。
3. 本轮做了什么代码、文档或结构改动，以及做了哪些验证。
4. 当前愿景目标完整度：
   - 用户层处于什么阶段
   - AI 与人协作层处于什么阶段
   - AI 治理层处于什么阶段
   - 运行基础 / 发布 / 兼容层处于什么阶段
   - 当前全局最大缺口是什么
5. 这次改动属于：架构增强 / 功能开发 / 场景闭环推进 / 文件解耦重构 / 运行基础或文档补全。
6. 它帮助了哪些方面：功能完整度、用户层、AI 与人协作层、AI 治理层、扩展性、兼容性、可靠性、稳定性、安全性。
7. 当前项目整体还缺什么，离完整闭环还差哪些关键环节。
8. 本轮是否陷入了局部细节优化而没有推动主链路；如果没有，说明为什么；如果有，指出并修正下一轮方向。
