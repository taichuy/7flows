# User Preferences

这里记录在开发过程中确认的“长期有效用户偏好”，用于指导后续实现、文档和 skill 调整。

## 使用原则

- 只记录稳定偏好，不记录一次性任务要求。
- 偏好需要能指导未来决策，而不是仅复述聊天内容。
- 如果某条偏好已经沉淀为仓库级规则或专项 skill 规则，需要在“落地动作”里说明同步位置。

## 记录模板

```md
## YYYY-MM-DD 偏好主题

- 偏好：
- 来源：
- 影响范围：
- 落地动作：
- 备注：
```
