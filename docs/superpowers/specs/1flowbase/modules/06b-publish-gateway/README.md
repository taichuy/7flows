# 06B 发布网关

日期：2026-04-14
状态：未来设计

## 讨论进度

- 状态：`future_design`
- 完成情况：已保留发布协议、`Publish Endpoint`、鉴权与对外适配的设计基线，但当前代码尚未形成实现闭环。
- 最后更新：2026-04-14 19:45 CST

## 本模块范围

- `Publish Endpoint`
- Flow 对外交付协议
- 应用级 API Key / Token 鉴权
- 原生协议与兼容协议适配
- 发布切换、回滚与线上流量语义

## 已确认设计基线

- `Publish Endpoint` 继续保留为 Flow 的对外交付物。
- 发布主目标仍然是稳定对外调用，而不是只停留在平台内编辑体验。
- 发布协议继续保留三类目标方向：
  - 原生协议
  - OpenAI 兼容协议
  - Claude 兼容协议
- 发布动作应保持原子语义：
  - 冻结当前 Draft
  - 生成新版本
  - 切换线上指针
- 历史版本 `Restore` 只恢复 Draft，不直接切线上流量。
- 对外鉴权继续先收敛为应用级 `API Key / Token`。

## 当前代码事实

- 当前代码库中 `publish-gateway` crate 仍只是骨架。
- 前端与后端都还没有正式的 `Publish Endpoint` 管理、对外发布协议或线上切换接口。
- 因此本模块当前不能作为“已实现能力”引用，只能作为未来开发专题。

## 与 04 / 05 / 06A 的边界

- `04` 负责 Flow 的编辑与版本化基础，不负责对外交付协议。
- `05` 负责未来编排运行时，不负责对外发布入口。
- `06A` 负责内部 API 文档，不负责外部发布契约。
- `06B` 只承接“把 Flow 变成可对外调用交付物”的那一层。

## 当前结论摘要

- `06B` 保留模块编号位置，但当前仍是未来设计。
- 后续进入开发时，应在 `04` 与 `05` 的稳定基线之上单独推进，不与内部 API 文档专题混写。
