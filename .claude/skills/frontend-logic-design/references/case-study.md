# 案例：看板交互一致性改造

> 从"用户觉得不对劲"到"四层信息深度模型完整落地"的全过程

## 背景

CloudDreamAI 数据管理系统的看板页面有 7 个维度卡片（待办事项、今日工作、项目概览、AI 洞察、风险预警、团队状态、工作进展），用户反馈：

- "今日工作里有个快速添加任务，但我添加的乱码也能 AI 执行，这有什么意义？"
- "AI 执行弹出来的东西，建议执行人显示'未指定'，点确认执行我不知道它能执行什么"
- "这些工作是 AI 整理的吧？为什么没有刷新按钮？"

表面上看是 3 个独立的 UI 问题，但诊断后发现根因是同一个：**缺少统一的信息架构规则**。

## Phase 1：识别问题类型

| 问题类型 | 发现 |
|---------|------|
| **深度不一致** | D01 全部 Tab 能点击看详情，待办/被驳回 Tab 不能点击 |
| **深度不一致** | D17 风险项完全不可点击 |
| **层级错位** | D10"快速添加任务"——概览层塞入了创建操作 |
| **层级错位** | D10"AI 执行"弹出表单——概览层塞入了复杂执行流程 |
| **入口缺失** | 没有任何卡片提供"查看全部"链接到对应的管理页面 |

## Phase 2：信息深度审计

逐个卡片标注每个交互元素的深度层：

| 卡片 | 元素 | 当前层级 | 应该的层级 | 问题 |
|------|------|---------|----------|------|
| D10 | 快速添加输入框 | L0 | L2 或 L3 | 概览层不该有创建操作 |
| D10 | AI 执行面板 | L0 | L3 | 概览层不该有复杂执行 |
| D10 | "去处理"→路由跳转 | L2 | L1 | 应该和点击行为一致（侧栏） |
| D01 | 待办 Tab 点击 | 无 | L1 | 应该能点击看详情 |
| D17 | 风险项点击 | 无 | L1 | 应该能点击看详情 |
| 全部 | "查看全部"入口 | 无 | L2 | 每个列表型卡片都应有 |

## Phase 3：MECE 检查

用户提出了一个关键问题：如果看板有侧栏，又有独立的 /tasks 页面，又有对话页面，这三者的定位是什么？互斥吗？

**错误的分类方式**（在 UI 容器维度做 MECE）：
```
侧栏放详情 / 页面放管理 / 对话放执行 → 看起来互斥
但用户体验不一致：有的有侧栏有的没有，有的有页面有的没有
```

**正确的分类方式**（在信息深度维度做 MECE）：
```
L0 概览（被动接收）→ L1 聚焦（单条详情）→ L2 管理（全量操作）→ L3 执行（AI 代劳）
容器只是实现手段，不是分类维度
```

互斥性验证：
- L0（多条摘要）≠ L1（单条详情）≠ L2（全量管理）≠ L3（AI 执行）✅

穷尽性验证：
- 用户的所有信息需求（概览/详情/管理/执行）全部覆盖 ✅

## Phase 5：修正方案

### 核心改动

1. **D10 去掉"快速添加任务"** — 创建操作属于 L2/L3，不属于 L0
2. **D10 "AI 执行"改为意图确认→跳转对话** — 复杂执行属于 L3
3. **D10 "去处理"改为打开 TaskDrawer** — 和点击行为一致（L1）
4. **D10/D07 加刷新按钮（force=true）** — AI 生成的内容需要重新生成入口
5. **所有列表型卡片加"查看全部 →"** — 统一 L2 入口
6. **D01/D17 补齐 clickable** — 统一 L1 行为
7. **D04 成员可点击跳转任务列表** — 补齐 L1→L2

### 基础设施

- `loadDimension(code, force?)` — 支持 force 参数
- `DimensionCard` 加 footer slot — 统一"查看全部"位置
- 新建 `AiIntentDialog` — 替代 `AiExecutePanel`

## 额外修复（验收找茬发现）

| 问题 | 根因 | 修复 |
|------|------|------|
| ChatView 不消费 message 参数 | ChatView 没有读 route.query | 加 useRoute，自动填入输入框 |
| TaskDrawer 负责人显示 UUID | 后端 getTask 没有 join 用户表 | 加 relations: ['assignee'] |
| 垃圾任务可创建 | CreateTaskDto 无 MinLength | 加 @MinLength(2) 校验 |
| AI 意图预填空白 | 后端 aiAction={} 空对象 | 基于已知信息填充 |

## 结果

- 7 个卡片的交互行为从各自为政变为统一规则
- 用户只需学一次："点击=侧栏，查看全部=页面，AI 执行=对话"
- 建立了可复用的四层信息深度模型，未来新增卡片只需遵循同一套规则
