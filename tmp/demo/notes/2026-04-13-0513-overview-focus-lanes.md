# 1Flowse Demo 本轮诊断与计划

时间：2026-04-13 05:13
范围：仅修改 `tmp/demo`
状态：已完成

## 本轮批判

- 桌面端 hero 右侧虽然已经不空，但上一版仍然像三张静态 signal card，回答的是“有哪几件事”，不是“下一步该去哪处理”。
- hero 左侧同时放 `进入编排 / 查看 API 契约 / 继续处理等待态` 三个按钮，会把主动作和二级路径压到同一层级，弱化了 `overview` 的唯一主入口规则。
- hero 下方四张数字摘要卡仍然太像通用 dashboard，不能真实表达当前 workspace 的问题域，也继续把桌面首屏拉得很长。

## 本轮修正

1. 保留 `进入编排` 作为 hero 唯一主动作。
2. 把二级路径收回到 hero 右侧的 `当前焦点` 面板：
   - `查看发布面` -> `API`
   - `打开 backlog` -> `Logs + Drawer`
   - `检查 host gap` -> `Monitoring`
3. 删除 `overview` 原来的数字摘要行，避免继续堆通用 stat card。
4. 保留 `最近运行摘要 / Published 与 Draft 明确分层 / 真实路由成熟度 / Embedded runtime snapshot` 四块作为 L0 主内容。

## 当前结果

- 桌面端 hero 终于开始回答“现在该看什么”，而不是继续展示一排无动作的摘要卡。
- 移动端首屏仍然只暴露一个主 CTA，但用户已经可以在同一屏看到三条明确的二级处理路径。
- `overview` 现在更接近 workspace 入口，而不是控制台首页 + 仪表盘数字的混合体。

## 仍然不够好的地方

- 桌面端 hero 左半边仍然偏空；根因不再是缺结构，而是当前焦点卡片的文案长度把整体高度拉起来了。
- 下一轮如果继续优化这里，不该再加 badge，而应该考虑：
  - 缩短焦点说明文案
  - 或把左侧空白换成一个更具体的 `run feed / micro canvas snapshot`
